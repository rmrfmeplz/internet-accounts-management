# 互联网账号管理 - Internet Accounts Management

一个**纯本地运行**、**无需联网**的个人互联网账号管理工具，解决个人账号数量过多、记不清账号归属 / 信息的核心痛点，所有数据本地
JSON 文件存储，无数据泄露风险，完全掌控自己的账号信息。

## 项目简介

本项目为个人本地桌面应用，专为个人互联网账号管理设计，无需部署服务器、无需联网使用，前端通过 Vue3 实现可视化操作界面，后端基于
NodeJS 提供本地接口服务，采用 JSON 文件替代传统数据库存储数据，结合 JWT 做本地身份校验，轻量化、易部署、无依赖外部服务，开箱即用。

## 核心特点

+ 📌 纯本地运行：全程无网络请求，所有操作、数据存储均在本地完成
+ 🔒 数据绝对安全：账号数据存储在本地 JSON 文件，无上传、无同步，避免数据泄露
+ 🛠 轻量化架构：NodeJS+Vue3+JSON+JWT 极简技术栈，无复杂配置，本地启动即可使用
+ 🖥 桌面端体验：基于 Electron 封装，支持桌面端运行，兼顾网页操作的便捷性
+ 📝 基础管理能力：满足账号的新增、查询、编辑、删除等核心管理需求
+ 🔑 本地身份校验：通过 JWT 实现接口权限验证，保障本地接口访问安全

## 技术栈

| 模块    | 技术 / 框架      | 说明                     |
|-------|--------------|------------------------|
| 前端界面  | Vue3         | 可视化账号管理操作界面            |
| 后端服务  | Node.js      | 提供本地 API 接口、数据处理       |
| 数据存储  | JSON 文件      | 替代数据库，本地轻量存储           |
| 桌面端封装 | Electron     | 网页应用转桌面端，本地运行          |
| 多服务启动 | concurrently | 一键启动后端 / 前端 / Electron |
| 身份验证  | JWT          | 本地接口权限校验、身份验证          |

## 项目结构

本项目采用前后端分离 + 桌面端封装的目录结构，各模块解耦，便于单独开发和维护：

```text
internet-accounts-management/ # 项目根目录
├── backend/                  # NodeJS后端（接口、JSON存储、JWT配置）
├── frontend/                 # Vue3前端（账号管理可视化界面）
├── electron/                 # Electron桌面端（本地桌面运行）
├── package.json              # 项目总配置（一键安装/启动脚本）
└── README.md                 # 项目说明文档
```

## 快速开始

### 环境要求

本地已安装 Node.js 和 npm（随 Node.js 自带），无需额外配置数据库、服务器等环境。

### 前置配置（必做）

项目基于 JWT 做本地接口身份校验，需手动配置密钥，步骤如下：

1. 进入项目的`backend`子目录
2. 新建`.env`配置文件（文件名固定，无多余后缀）
3. 在`.env`文件中添加如下配置，替换自定义密钥：

```text
JWT_SECRET=your_custom_secret_key
```

✅ 说明：将your_custom_secret_key替换为自定义随机字符串（建议包含字母、数字、特殊符号，越长越安全），该密钥仅本地使用，无泄露风险。

### 运行步骤（所有命令均在项目根目录执行）

1, 进入项目根目录

```text
cd internet-accounts-management
```

2, 一键安装所有依赖（必做）
自动安装根目录 + backend + frontend + electron 所有模块的依赖，无需逐个目录安装：

```text
npm run install:all
```

3, 一键启动项目
自动同时启动后端、前端、Electron 桌面端，启动成功后会自动打开桌面应用窗口：

```text
npm start
```

### 停止项目

在启动项目的终端中，按下 Ctrl + C 并按提示确认（输入 y / 回车），即可一次性停止所有服务。

## 数据存储说明

1. 所有数据均存储在backend/data目录下的 JSON 文件中，无数据库依赖
2. 数据的读写操作均由 NodeJS 后端接口统一处理，前端仅通过接口交互，不直接操作本地文件
3. 纯本地存储：JSON 文件保存在本地电脑，无任何联网上传、同步行为，数据完全由个人掌控
4. 建议定期备份该 JSON 文件，避免电脑重装、文件误删导致数据丢失

## 注意事项

1. 本项目为个人本地使用开发，未做生产环境优化，仅适配单人工况，不支持多用户
2. 启动失败优先排查：终端是否在项目根目录、依赖是否安装完整（可重新执行npm run install:all）、backend/.env文件是否配置正确
3. .env文件包含 JWT 核心密钥，不要提交到 Git/GitHub 等版本库
4. 启动后，若 Electron 窗口未自动打开，或打开后无任何内容显示，可先检查终端是否出现红色报错信息；也可直接重新执行 npm start
   命令尝试解决。

说明：本项目为纯本地自用工具，无任何远程服务依赖，所有代码和数据均保存在本地电脑，可根据个人需求自由二次开发。
